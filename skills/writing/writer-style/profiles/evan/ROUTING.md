# ROUTING — evan pack

Two steps. **Mode first** (presentation job), then **secondary craft** (optional).
Primary **evan** is always on. Topic (math / data / markets / agents) never chooses the mode.

## Step 1 — Presentation mode

Full cards: [`modes.md`](modes.md).

| Dominant job (domain-agnostic) | Mode |
|--------------------------------|------|
| Report a measured/observed result | **findings-note** |
| Transfer production scars / what I built | **field-log** |
| Digest a paper, book, or theory in public | **claim-diary** |
| Argue which system/design wins | **systems-essay** |
| Same task across tools/approaches | **bake-off** |
| Reader must execute a path (narrative how-to) | **walkthrough** |
| Pure procedure / runbook / reference | → **writing-docs** skill (not a voice mode) |
| New lens that makes a structure legible | **concept-map** |
| Explain an artifact; constraint→technique | **design-brief** |
| Public partial result; questions are the point | **forum-fragment** |
| Unclear technical default | **findings-note** |

## Step 2 — Secondary craft (optional)

Profiles at `../kaue/secondary/` — **idiolect stripped**. Cap: 1 backbone (+ optional helius guest).

| Dominant craft need | Lead secondary |
|---------------------|----------------|
| Show artifact, tables, checkable method | **helius** |
| Derive deep tradeoffs / mechanism | **vitalik** |
| Broad thesis framing | **balaji** |
| Long incentive/market stamina argument | **hayes** |
| Compress to one honest model | **hotz** |
| Field-log / pure operator notes | **evan alone** |

### Mode × secondary defaults

| Mode | Typical secondary |
|------|-------------------|
| findings-note | helius (or alone) |
| field-log | alone (helius guest if heavy artifacts) |
| claim-diary | vitalik light, or alone |
| systems-essay | hayes or balaji light |
| bake-off | helius |
| walkthrough | helius |
| concept-map | alone or vitalik light |
| design-brief | helius or vitalik |
| forum-fragment | alone |

**Never** let secondary craft erase: chosen mode structure, evidence objects, open gaps, or frozen facts.

## Hard fails

- Using **field-log** scar cosplay for a pure claim-diary  
- Using **findings-note** TL;DR theater with no actual findings  
- Routing by topic (“it’s math so academic”) instead of job  
- Soft “In conclusion we have shown…” restatement of the open
