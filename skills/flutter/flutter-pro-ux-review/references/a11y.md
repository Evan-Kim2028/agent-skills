# Pass: a11y

Accessibility failures are polish failures. Unusable primary controls → **Critical**.

### Touch targets large enough

`min-touch-target` · Critical on primary actions; Noticeable on secondary · Effort: S  
Fix in one phrase: interactive targets ≥ ~48×48 dp (padding counts); don’t rely on 16dp icons alone.

**Detect:** `IconButton` with tiny constraints, custom `GestureDetector` icons without `minimumSize` / padding, dense chips that are the only control for a primary action.

**Hunt:** icon-only actions in app bars and lists; check `constraints`, `padding`, `minimumSize` on `ButtonStyle`.

**Why:** mis-taps and exclusion of users with motor difficulty.

**Gotchas:** dense data tables may use smaller secondary targets if a larger hit area still exists; document exceptions.

### Semantics on icon-only and image buttons

`semantics-tappable` · Critical on primary; Noticeable on secondary · Effort: S  
Fix in one phrase: every tappable without visible text gets a `Semantics` label / `tooltip` / `IconButton` semantic label.

**Detect:** `IconButton` / `InkWell` / `GestureDetector` with only an `Icon` and no `tooltip`, `semanticsLabel`, or wrapping `Semantics(label:, button: true)`.

**Hunt:** grep `IconButton|Icons\.` in onTap widgets; check for labels.

**Why:** screen reader users hear “button” with no name.

**Gotchas:** decorative non-interactive icons should be excluded via `ExcludeSemantics` / `Semantics(excludeSemantics: true)` — don’t flag those as missing labels if they aren’t tappable.

### Text scale must not destroy layout

`text-scale-survival` · Critical if primary UI overflows/clips at large scale; Noticeable if cramped · Effort: M  
Fix in one phrase: prefer reflow (`Wrap`/`Flexible`/`maxLines`+ellipsis with the full text still available) up to ~200% text scale over clamping `textScaler`; clamping to dodge overflow defeats the user's own setting.

**Detect:** fixed-height rows with multi-line labels, `Text` in tight `Row`s without `Flexible`/`Expanded`, hardcoded heights that clip at 1.3–2.0 text scale. Also flag `textScaler:`/`TextScaler.clamp` calls used purely to prevent overflow.

**Hunt:** dense list tiles, bottom nav labels, buttons with fixed height; grep `height:` near `Text`/`ListTile`; grep `textScaler|TextScaler.clamp|MediaQuery.textScalerOf` and check whether the clamp exists to fix layout instead of a deliberate readability ceiling.

**Why:** system text size is an a11y setting; clipping or silently capping it below what the user chose defeats the accommodation they turned on.

**Gotchas:** clamping is a last resort, not a first fix — reflow first. If a clamp is truly needed for a complex tool, never clamp below 1.3, and document why reflow wasn't viable. Still flag broken primary auth/home at default large sizes regardless of clamp presence.

### Status not by color alone

`status-not-color-only` · Noticeable · Effort: S  
Fix in one phrase: success/error/warning include icon or text, not only red/green.

**Detect:** status rows that change only `color:` / container color with no icon/text change.

**Hunt:** error/success banners, validation UI, order status chips.

**Why:** color-blind and low-vision users miss the state.

**Gotchas:** brand accents on already-labeled text are fine.

---

### Contrast on custom surfaces

`contrast-custom-surfaces` · Noticeable (Critical if body text unreadable) · Effort: M  
Fix in one phrase: text on custom/brand colors meets readable contrast; don’t use low-contrast grey on grey.

**Detect:** light grey text on white or colored fills; white text on pale brand colors in widgets with hardcoded `Color(0x…)`.

**Hunt:** hardcoded `Color(` / `Colors.grey` text styles on non-theme surfaces.

**Why:** unreadable UI is a functional bug.

**Gotchas:** full WCAG lab measurement isn’t required in static review — flag clear failures; theme-driven `onSurface` usage is usually OK.
